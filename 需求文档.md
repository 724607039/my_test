# 低代码可拖拽组件系统需求文档

## 1. 项目概述

本项目旨在开发一套低代码可拖拽实现的组件系统，用户可以通过拖拽方式构建内容，并支持生成Word文件和打印功能。系统将提供直观的可视化界面，降低用户的技术门槛，实现快速内容创作和导出。

## 4. 技术方案

### 4.1 技术栈

| 技术 | 版本 | 用途 |
|-----|------|------|
| Vue | 3.4+ | 前端框架 |
| Vite | 4.4+ | 构建工具 |
| SortableJS | 1.15+ | 拖拽功能 |
| Docxtemplater | 3.42+ | Word文档生成 |
| PizZip | 3.1+ | 处理ZIP文件 |
| FileSaver | 2.0+ | 文件下载 |

### 4.2 系统架构

- **前端架构**：基于Vue 3的单页应用，采用组件化设计
- **数据流**：使用Vue的响应式数据管理组件状态
- **Word生成**：客户端生成，无需后端服务

### 4.3 核心流程

1. 用户从组件库拖拽组件到画布
2. 系统在画布上渲染组件
3. 用户编辑组件属性
4. 用户点击生成Word按钮
5. 系统收集画布组件数据
6. 系统生成Word文档并触发下载
7. 用户点击打印按钮，系统调用浏览器打印功能

## 5. 实现细节

### 5.1 拖拽功能实现

- 使用HTML5原生拖拽API实现组件拖拽
- 拖拽开始时存储组件信息
- 拖拽结束时在画布上创建组件实例

### 5.2 组件管理

- 维护画布组件数组，包含组件类型和属性
- 支持组件的添加、删除、属性更新
- 实时更新画布显示

### 5.3 Word生成实现

- 使用PizZip创建Word文档结构
- 使用Docxtemplater处理文档内容
- 支持文本、图片、表格的Word格式转换
- 处理图片的base64编码和嵌入

### 5.4 打印功能实现

- 调用浏览器的window.print()方法
- 优化打印样式，确保打印效果美观

## 6. 使用说明

### 6.1 基本操作

1. **添加组件**：从左侧组件库拖拽组件到中间画布
2. **编辑组件**：点击画布上的组件，在右侧属性面板编辑属性
3. **删除组件**：点击组件右上角的删除按钮
4. **生成Word**：点击顶部操作栏的"生成Word"按钮
5. **打印**：点击顶部操作栏的"打印"按钮

### 6.2 组件使用

- **文本组件**：直接在属性面板编辑文本内容
- **图片组件**：点击"上传本地图片"按钮选择图片，或输入图片URL
- **表格组件**：使用表格上方的按钮添加/删除行列，点击单元格编辑内容
